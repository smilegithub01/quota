## 2. 模块划分原则

### 2.1 高内聚低耦合

- **高内聚**：模块内部的功能紧密相关，共同完成一个明确的业务目标
- **低耦合**：模块之间的依赖关系尽可能少，通过标准接口进行交互
- **实现方式**：
  - 按业务领域划分模块
  - 使用依赖注入解耦
  - 通过事件驱动实现异步通信

### 2.2 单一职责

- **原则**：每个模块只负责一个明确的业务领域
- **实现方式**：
  - 模块职责清晰明确
  - 避免功能交叉
  - 支持独立开发和测试

### 2.3 接口标准化

- **原则**：所有模块接口遵循统一的规范
- **实现方式**：
  - 使用RESTful API规范
  - 统一请求响应格式
  - 统一错误码定义
  - 统一版本管理

### 2.4 可扩展性

- **原则**：模块设计支持灵活扩展
- **实现方式**：
  - 采用插件化架构
  - 使用配置化管理
  - 支持模块热插拔

| 额度释放服务 | 处理还款/终止时的额度返还 | 释放计算、穿透返还、明细记录 |
| 额度终止服务 | 处理额度到期/主动/强制终止 | 终止校验、数据清理、归档处理 |

| 额度充足性校验服务 | 校验额度是否充足 | 单层校验、穿透校验、批量校验 |

#### 3.2.3 额度控制模块

| 服务名称 | 职责描述 | 核心功能 |
|---------|---------|---------|
| 额度锁定服务 | 临时锁定额度 | 锁定申请、超时释放、锁定查询 |
| 额度占用服务 | 正式占用额度 | 单占处理、多占处理、穿透占用 |

### 5.3 错误码定义

#### 5.3.1 错误码结构设计

错误码采用6位编码格式：**业务领域(2位) + 模块(2位) + 具体错误(2位)**

| 编码段 | 含义 | 示例 |
|--------|------|------|
| 第1-2位 | 业务领域 | QT-额度、SY-系统、SF-审批、RZ-风险 |
| 第3-4位 | 模块 | QY-查询、KZ-控制、YZ-预警、SJ-审计 |
| 第5-6位 | 具体错误 | 01-成功、02-失败、03-异常 |

#### 5.3.2 错误级别定义

| 错误级别 | 编码前缀 | 说明 | 处理方式 |
|----------|----------|------|----------|
| INFO | 9 | 提示性信息 | 记录日志，可正常返回 |
| WARN | 8 | 警告信息 | 记录日志，返回警告信息 |
| ERROR | 0 | 错误信息 | 记录日志，返回错误信息 |

### 6.2 依赖说明

#### 6.2.1 额度查询服务依赖

- **依赖服务**：基础限额服务、集团限额服务、系统管理服务
- **依赖组件**：Redis集群、MySQL集群
- **依赖说明**：额度查询服务依赖基础限额服务和集团限额服务提供额度数据，依赖Redis集群提供缓存支持

#### 6.2.2 额度控制服务依赖

- **依赖服务**：基础限额服务、集团限额服务、实时控制服务、审计追踪服务
- **依赖组件**：Redis集群、MySQL集群、RabbitMQ集群
- **依赖说明**：额度控制服务依赖基础限额服务和集团限额服务提供额度数据，依赖实时控制服务提供实时校验，依赖审计追踪服务记录操作日志

#### 6.2.3 风险预警服务依赖

- **依赖服务**：额度查询服务、阈值预警服务
- **依赖组件**：Redis集群、MySQL集群、RabbitMQ集群
- **依赖说明**：风险预警服务依赖额度查询服务提供额度数据，依赖阈值预警服务提供预警规则

#### 6.2.4 额度计算服务依赖

- **依赖服务**：基础限额服务、集团限额服务
- **依赖组件**：MySQL集群
- **依赖说明**：额度计算服务依赖基础限额服务和集团限额服务提供基础数据

#### 6.2.5 审计追踪服务依赖

- **依赖服务**：系统管理服务
- **依赖组件**：MySQL集群、RabbitMQ集群
- **依赖说明**：审计追踪服务依赖系统管理服务提供用户信息

#### 6.2.6 监控大屏服务依赖

- **依赖服务**：额度查询服务、风险预警服务
- **依赖组件**：Redis集群
- **依赖说明**：监控大屏服务依赖额度查询服务提供额度数据，依赖风险预警服务提供预警信息

## 7. 数据流设计

### 7.1 额度查询数据流

```
用户请求 → API网关 → 额度查询服务 → Redis缓存 → MySQL数据库
                                    ↓
                              返回额度数据
```

### 7.2 额度占用数据流

```
用户请求 → API网关 → 额度控制服务 → 额度校验 → 额度锁定
                                    ↓
                              额度占用
                                    ↓
                              数据库更新
                                    ↓
                              审计日志记录
                                    ↓
                              返回操作结果
```

### 7.3 额度释放数据流

```
用户请求 → API网关 → 额度控制服务 → 额度校验 → 额度释放
                                    ↓
                              数据库更新
                                    ↓
                              审计日志记录
                                    ↓
                              返回操作结果
```
