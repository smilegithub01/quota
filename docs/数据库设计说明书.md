## 2. 数据库设计原则

### 2.1 规范化设计

#### 2.1.1 第一范式（1NF）

- 每个字段都是不可分割的原子值
- 消除重复字段

#### 2.1.2 第二范式（2NF）

- 满足第一范式
- 非主键字段完全依赖于主键
- 消除部分依赖

#### 2.1.3 第三范式（3NF）

- 满足第二范式
- 非主键字段不传递依赖于主键
- 消除传递依赖

### 2.2 性能优化设计

#### 2.2.1 索引优化

- 为查询字段建立索引
- 为外键字段建立索引
- 为常用查询条件建立复合索引
- 避免过度索引

#### 2.2.2 查询优化

- 避免全表扫描
- 避免使用SELECT *
- 避免使用子查询
- 使用分页查询

#### 2.2.3 表设计优化

- 合理选择字段类型
- 使用适当的数据长度
- 使用NOT NULL约束
- 使用默认值

### 2.3 安全性设计

#### 2.3.1 访问控制

- 使用最小权限原则
- 分离读写权限
- 定期审计权限

#### 2.3.2 数据加密

- 敏感字段加密存储
- 使用AES-256加密
- 密钥安全管理

#### 2.3.3 数据备份

- 定期备份数据
- 备份数据加密
- 备份数据异地存储

### 2.4 可扩展性设计

#### 2.4.1 分库分表

- 按业务领域分库
- 按数据量分表
- 支持动态扩容

#### 2.4.2 读写分离

- 主库负责写操作
- 从库负责读操作
- 支持多从库

## 3. 数据库架构设计

### 3.1 数据库选型

#### 3.1.1 关系型数据库：MySQL 8.0

**选择理由**：
- 开源免费，性能优秀
- 支持事务，ACID特性
- 支持主从复制
- 支持分库分表
- 银行业广泛使用

**核心特性**：
- InnoDB存储引擎
- 支持事务
- 支持外键约束
- 支持索引优化
- 支持主从复制

### 3.2 数据库部署架构

#### 3.2.1 主从复制架构

```
┌─────────────────────────────────────────────────────────────┐
│                      主库（Master）                            │
│  负责写操作                                                    │
└─────────────────────────────────────────────────────────────┘
                              ↓
                              │ binlog
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      从库1（Slave1）                           │
│  负责读操作                                                    │
└─────────────────────────────────────────────────────────────┘
                              ↑
                              │ binlog
                              ↑
┌─────────────────────────────────────────────────────────────┐
│                      从库2（Slave2）                           │
│  负责读操作                                                    │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 数据库分库分表

#### 3.3.1 分库策略

按业务领域分库：

- quota_db：额度管理库
- mapping_db：业务映射库
- risk_db：风险预警库
- audit_db：审计追踪库
- system_db：系统管理库

#### 3.3.2 分表策略

按数据量分表：

- group_quota：集团限额表（数据量小，不分表）
- customer_quota：客户额度表（数据量大，按customer_id分表）
- contract_occupancy：合同占用表（数据量大，按contract_no分表）
- operation_log：操作日志表（数据量大，按create_time分表）

### 3.4 读写分离

#### 3.4.1 读写分离架构

- 主库：负责写操作（INSERT、UPDATE、DELETE）
- 从库：负责读操作（SELECT）
- 中间件：ShardingSphere

#### 3.4.2 读写分离配置

```yaml
spring:
  shardingsphere:
    datasource:
      names: master,slave1,slave2
      master:
        type: com.zaxxer.hikari.HikariDataSource
        driver-class-name: com.mysql.cj.jdbc.Driver
        jdbc-url: jdbc:mysql://master:3306/quota_db
        username: root
        password: password
      slave1:
        type: com.zaxxer.hikari.HikariDataSource
        driver-class-name: com.mysql.cj.jdbc.Driver
        jdbc-url: jdbc:mysql://slave1:3306/quota_db
        username: root
        password: password
      slave2:
        type: com.zaxxer.hikari.HikariDataSource
        driver-class-name: com.mysql.cj.jdbc.Driver
        jdbc-url: jdbc:mysql://slave2:3306/quota_db
        username: root
        password: password
    rules:
      readwrite-splitting:
        data-sources:
          quota_db:
            static-strategy:
              write-data-source-name: master
              read-data-source-names: slave1,slave2
            load-balancer-name: round_robin
        load-balancers:
          round_robin:
            type: ROUND_ROBIN
```

## 5. 表结构设计

### 5.1 额度生命周期表

#### 5.1.1 额度调整记录表（quota_adjustment）

```sql
CREATE TABLE `quota_adjustment` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键',
  `adjustment_no` VARCHAR(50) NOT NULL COMMENT '调整编号',
  `quota_id` BIGINT NOT NULL COMMENT '额度ID',
  `quota_type` VARCHAR(20) NOT NULL COMMENT '额度类型：GROUP-集团，CUSTOMER-客户，APPROVAL-批复',
  `adjustment_type` VARCHAR(20) NOT NULL COMMENT '调整类型：TEMPORARY-临时，PERMANENT-永久，BATCH-批量',
  `adjustment_amount` DECIMAL(20,4) NOT NULL COMMENT '调整金额（正数为增加，负数为减少）',
  `before_amount` DECIMAL(20,4) NOT NULL COMMENT '调整前金额',
  `after_amount` DECIMAL(20,4) NOT NULL COMMENT '调整后金额',
  `reason` VARCHAR(500) NOT NULL COMMENT '调整原因',
  `effective_time` DATETIME COMMENT '生效时间',
  `expiry_time` DATETIME COMMENT '失效时间（临时调整）',
  `status` VARCHAR(20) NOT NULL DEFAULT 'PENDING' COMMENT '状态：PENDING-待审批，APPROVED-已审批，COMPLETED-已完成，REJECTED-已拒绝',
  `applicant` VARCHAR(50) NOT NULL COMMENT '申请人',
  `approver` VARCHAR(50) COMMENT '审批人',
  `approve_time` DATETIME COMMENT '审批时间',
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_adjustment_no` (`adjustment_no`),
  KEY `idx_quota_id` (`quota_id`),
  KEY `idx_quota_type` (`quota_type`),
  KEY `idx_adjustment_type` (`adjustment_type`),
  KEY `idx_status` (`status`),
  KEY `idx_create_time` (`create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='额度调整记录表';
```

### 8.1 额度占用存储过程

```sql
DELIMITER $$

CREATE PROCEDURE `sp_occupy_quota`(
    IN p_contract_no VARCHAR(50),
    IN p_approval_quota_sub_id BIGINT,
    IN p_occupancy_amount DECIMAL(20,4),
    IN p_operator VARCHAR(50),
    OUT p_result_code VARCHAR(20),
    OUT p_result_message VARCHAR(500)
)
BEGIN
    DECLARE v_available_quota DECIMAL(20,4);
    DECLARE v_approval_quota_id BIGINT;
    DECLARE v_customer_quota_sub_id BIGINT;
    DECLARE v_customer_quota_id BIGINT;
    DECLARE v_group_quota_sub_id BIGINT;
    DECLARE v_group_quota_id BIGINT;
    DECLARE v_error_code CHAR(5) DEFAULT '00000';
    DECLARE v_error_msg TEXT;
    
    DECLARE CONTINUE HANDLER FOR SQLEXCEPTION 
    BEGIN
        GET DIAGNOSTICS CONDITION 1 v_error_code = RETURNED_SQLSTATE, v_error_msg = MESSAGE_TEXT;
        SET p_result_code = '000001';
        SET p_result_message = CONCAT('系统错误：', v_error_msg);
        ROLLBACK;
    END;
    
    START TRANSACTION;
    
    SELECT available_quota, approval_quota_id INTO v_available_quota, v_approval_quota_id
    FROM approval_quota_sub
    WHERE id = p_approval_quota_sub_id AND status = 'ENABLED'
    FOR UPDATE;
    
    IF v_available_quota IS NULL THEN
        SET p_result_code = '000005';
        SET p_result_message = '授信产品细项不存在或已失效';
        ROLLBACK;
        LEAVE sp_occupy_quota;
    END IF;
    
    IF v_available_quota < p_occupancy_amount THEN
        SET p_result_code = '000006';
        SET p_result_message = CONCAT('额度不足，可用额度：', v_available_quota, '，占用金额：', p_occupancy_amount);
        ROLLBACK;
        LEAVE sp_occupy_quota;
    END IF;
    
    UPDATE approval_quota_sub
    SET used_quota = used_quota + p_occupancy_amount,
        available_quota = available_quota - p_occupancy_amount,
        update_time = NOW(),
        update_by = p_operator
    WHERE id = p_approval_quota_sub_id;
    
    INSERT INTO contract_occupancy (contract_no, approval_quota_sub_id, occupancy_amount, status, create_by, update_by)
    VALUES (p_contract_no, p_approval_quota_sub_id, p_occupancy_amount, 'OCCUPIED', p_operator, p_operator);
    
    SELECT customer_quota_sub_id INTO v_customer_quota_sub_id
    FROM approval_quota
    WHERE id = v_approval_quota_id;
    
    UPDATE customer_quota_sub
    SET used_quota = used_quota + p_occupancy_amount,
        available_quota = available_quota - p_occupancy_amount,
        update_time = NOW(),
        update_by = p_operator
    WHERE id = v_customer_quota_sub_id;
    
    SELECT customer_quota_id INTO v_customer_quota_id
    FROM customer_quota_sub
    WHERE id = v_customer_quota_sub_id;
    
    UPDATE customer_quota
    SET used_quota = used_quota + p_occupancy_amount,
        available_quota = available_quota - p_occupancy_amount,
        update_time = NOW(),
        update_by = p_operator
    WHERE id = v_customer_quota_id;
    
    COMMIT;
    
    SET p_result_code = '000000';
    SET p_result_message = '额度占用成功';
    
END$$
```

```sql
CREATE PROCEDURE `sp_release_quota`(
    IN p_contract_no VARCHAR(50),

    IN p_reason VARCHAR(500),

        SET p_result_message = CONCAT('系统错误：', v_error_msg);
        ROLLBACK;

    START TRANSACTION;
    
    SELECT approval_quota_sub_id, occupancy_amount INTO v_approval_quota_sub_id, v_occupancy_amount
    FROM contract_occupancy
    WHERE contract_no = p_contract_no AND status = 'OCCUPIED'

        ROLLBACK;
        LEAVE sp_release_quota;

    SET status = 'RELEASED',
        update_time = NOW(),
        update_by = p_operator
```
