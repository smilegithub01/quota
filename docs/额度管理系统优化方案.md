## 1. 系统现状分析

### 1.1 现有系统架构

当前额度管理系统采用六级额度管控体系：

```
第一层：集团限额 (group_quota)
    ↓
第二层：集团限额细项 (group_quota_sub)
    ↓
第三层：单一客户额度 (customer_quota)
    ↓
第四层：客户额度细项 (customer_quota_sub)
    ↓
第五层：业务授信批复 (approval_quota)
    ↓
第六层：业务合同占用 (contract_occupancy)
```

### 1.2 现有核心功能模块

| 模块 | 当前实现状态 | 技术实现 |
|------|-------------|----------|
| 额度配置管理 | 已实现 | GroupQuotaService, CustomerQuotaService |
| 额度实时管控 | 部分实现 | QuotaControlService |
| 额度锁定/释放 | 已实现 | Redisson分布式锁 |
| 额度占用/归还 | 已实现 | 单层占用逻辑 |
| 风险预警 | 已实现 | RiskWarningService |
| 审计追踪 | 已实现 | AuditLogService |

## 2. 差异对比分析

### 2.1 额度校验机制差异对比

| 对比维度 | 当前实现 | 目标要求 | 差异分析 |
|----------|----------|----------|----------|
| 校验层级 | 单层校验（客户/批复级别） | 六级递进校验 | 缺少集团→客户→批复→合同的完整链路校验 |
| 校验顺序 | 无严格顺序 | 从高层级到低层级递进 | 需要实现层级递进校验流程 |
| 错误信息 | 简单错误提示 | 包含层级、原因、规则依据 | 需要增强错误信息结构 |
| 规则配置 | 硬编码在代码中 | 可配置化规则库 | 需要设计规则配置表和规则引擎 |
| 版本管理 | 无 | 支持灰度发布和回滚 | 需要设计规则版本管理机制 |
| 业务场景 | 通用校验 | 按业务类型差异化校验 | 需要支持流动资金贷款、项目贷款等不同场景 |

### 2.2 额度占用逻辑差异对比

| 对比维度 | 当前实现 | 目标要求 | 差异分析 |
|----------|----------|----------|----------|
| 占用层级 | 单层占用（批复级别） | 六级穿透占用 | 缺少向上穿透更新机制 |
| 事务管理 | 单机事务 | 分布式事务（TCC/SAGA） | 需要引入分布式事务框架 |
| 状态管理 | 简单状态（OCCUPIED/RELEASED） | 预占/确认/释放全生命周期 | 需要增加预占状态和状态转换逻辑 |
| 临时占用 | 无 | 支持24小时临时占用 | 需要设计临时占用机制 |
| 多币种 | 单币种（CNY） | 多币种支持 | 需要设计汇率管理和多币种转换 |
| 操作轨迹 | 简单日志 | 完整操作轨迹（操作人、时间、IP、原因） | 需要增强审计日志记录 |

### 2.3 统一授信整合差异对比

| 对比维度 | 当前实现 | 目标要求 | 差异分析 |
|----------|----------|----------|----------|
| 监管规则 | 部分实现 | 完整映射监管要求 | 需要建立法规要求与系统功能映射表 |
| 数据同步 | 无 | 双向实时同步（延迟≤5秒） | 需要设计数据同步机制 |
| 预警机制 | 单一阈值 | 多级别预警（80%/90%/95%） | 需要增强预警规则配置 |
| 通知渠道 | 系统内消息 | 短信/邮件/系统消息多渠道 | 需要集成消息通知服务 |

### 2.4 系统实现规范差异对比

| 对比维度 | 当前实现 | 目标要求 | 差异分析 |
|----------|----------|----------|----------|
| 数据模型 | 基本完整 | 完整ER图和数据字典 | 需要补充文档 |
| 事务回滚 | 基本实现 | 本地消息表/消息队列保障 | 需要增强最终一致性保障 |
| 链路追踪 | 无 | 全链路可视化展示 | 需要设计链路追踪功能 |
| 审计日志 | 基本实现 | 7年保存、多条件查询、Excel导出 | 需要增强审计功能 |

## 4. 额度生命周期管控优化方案

### 4.1 额度生命周期管理现状

当前系统对额度生命周期的管理较为分散，缺乏统一的管控机制：

| 生命周期阶段 | 当前实现状态 | 存在问题 |
|-------------|-------------|---------|
| 额度创建 | 已实现 | 缺少合规性校验和层级约束校验 |
| 额度调整 | 部分实现 | 缺少临时调整、批量调整功能 |
| 额度使用 | 已实现 | 缺少预占模式支持 |
| 额度冻结/解冻 | 已实现 | 缺少条件冻结和联动处理 |
| 额度释放 | 已实现 | 缺少穿透式返还机制 |
| 额度终止 | 未实现 | 缺少到期终止和强制终止功能 |

### 4.2 额度生命周期管控架构设计

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        额度生命周期管控中心                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                        生命周期状态机                                │   │
│  │  PENDING(待启用) → ACTIVE(正常) ↔ FROZEN(冻结) → TERMINATED(已终止) │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐      │
│  │ 创建服务 │   │ 调整服务 │   │ 使用服务 │   │ 冻结服务 │   │ 终止服务 │      │
│  └────┬────┘   └────┬────┘   └────┬────┘   └────┬────┘   └────┬────┘      │
│       │             │             │             │             │            │
│       ↓             ↓             ↓             ↓             ↓            │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                        生命周期事件总线                              │   │
│  │  事件发布 → 事件订阅 → 事件处理 → 事件持久化 → 事件通知             │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                        规则引擎集成                                  │   │
│  │  合规规则 │ 层级约束规则 │ 联动规则 │ 状态转换规则                   │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.3 核心优化内容

#### 4.3.1 额度创建优化

增加合规性校验、层级约束校验和额度编号自动生成功能。

#### 4.3.2 额度调整优化

支持临时调整、永久调整、批量调整三种模式，实现联动更新机制。

#### 4.3.3 额度冻结/解冻优化

支持全额冻结、部分冻结、条件冻结三种模式，实现联动处理。

#### 4.3.4 额度终止优化

支持到期终止、主动终止、强制终止三种模式，实现终止前检查和数据归档。

    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '汇率ID',
    currency_code VARCHAR(10) NOT NULL COMMENT '货币代码: CNY/USD/EUR/JPY等',
    currency_name VARCHAR(50) NOT NULL COMMENT '货币名称',
    rate_to_cny DECIMAL(20,8) NOT NULL COMMENT '对人民币汇率',
    rate_source VARCHAR(50) NOT NULL COMMENT '汇率来源: REUTERS-路透, PBOC-央行, MANUAL-手工',
    effective_date DATE NOT NULL COMMENT '生效日期',
    effective_time DATETIME NOT NULL COMMENT '生效时间',
    status VARCHAR(20) NOT NULL DEFAULT 'ACTIVE' COMMENT '状态: ACTIVE-有效, INACTIVE-无效',
    create_by VARCHAR(50) COMMENT '创建人',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    UNIQUE KEY uk_currency_date (currency_code, effective_date),
    INDEX idx_currency_code (currency_code),
    INDEX idx_effective_date (effective_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='汇率配置表';
